Transitions

key idea: whatever is having the transition done to it would have this in its style sheet

  • transition-property: "whatever property you want to change";
  • transition-duration: "how long you want change to take";
  • transition-delay: "how long before change starts";

Dev Toolchain

key idea: use whenever adding packages for projects

Setup

  • run npm init in terminal to intialize npm project
  • install packages you'll be using for project
  • create html and js files and do some test code
  • install npm install --save-dev @web/dev-server
  • add following to scrips in package.json: "start": "web-dev-server --node-resolve --open --watch"
  • run npm run start in terminal to run app locally

Build with Rollup

  • install rollup by npm install rollup --save-dev in terminal
  • npm install @rollup/plugin-node-resolve --save-dev in terminal
  • npm install rollup-plugin-copy --save-dev in terminal
  • create rollup.config.js file
  • add "build": "rollup --config" to scripts in package.json
  • run npm run build in terminal to build app
  • create .gitignore file with node_modules/ dist/ in it
  • push to github to save work

Deploy with GH-Pages

  • install gh-pages by doing npm install gh-pages --save-dev in terminal
  • add "deploy": "npm run build && gh-pages -d dist" into package.json scripts
  • run npm run deploy to deploy app
  • to see app visit: (gitHub username).github.io/(repo name)
